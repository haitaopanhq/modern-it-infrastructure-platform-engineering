# 09-04: 一图看懂 GPU 调度

## 核心观点
GPU 调度比 CPU 调度更复杂，需要考虑显存、拓扑和共享。

## GPU 调度挑战

```
┌─────────────────────────────────────────────────────────────┐
│                    GPU 调度                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   调度维度:                                                 │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  1. 显存调度                                         │  │
│   │     • 显存碎片                                        │  │
│   │     • 显存超时                                        │  │
│   │     • 多租户显存隔离                                  │  │
│   └─────────────────────────────────────────────────────┘  │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  2. 拓扑调度                                         │  │
│   │     • NVLink (单机多 GPU)                           │  │
│   │     • RDMA (多机多 GPU)                              │  │
│   │     • 带宽感知                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  3. 共享调度                                         │  │
│   │     • 时间片共享                                     │  │
│   │     • 算力分割 (MIG)                                 │  │
│   │     • 抢占调度                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   工具: TimeSlicing, MIG, Gang Scheduling                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
