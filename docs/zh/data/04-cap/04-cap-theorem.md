# 06-04: 一图看懂 CAP 理论

## 核心观点
CAP 理论不是三选二，而是在不同场景下权衡可用性和一致性。

## CAP 权衡

```
┌─────────────────────────────────────────────────────────────┐
│                      CAP 理论                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│        Consistency (一致性)                                 │
│              │                                              │
│              │        CA                                    │
│              │    ┌────────┐                                │
│              ├───►│  传统  │                                │
│              │    │  RDBMS │                                │
│              │    └────────┘                                │
│              │                                              │
│     ┌────────┼────────────────────────────────┐            │
│     │        │                                │            │
│     │   CP   │                                │   AP       │
│     │┌──────┐│                                │┌────────┐ │
│     ││DNS   ││                                ││ Dynamo │ │
│     ││Zoo   ││                                ││Cassandra│ │
│     ││Keeper││                                ││  etcd  │ │
│     │└──────┘│                                │└────────┘ │
│     │        │                                │            │
│     └────────┴────────────────────────────────┘            │
│              │                                              │
│              │      Availability (可用性)                  │
│              │                                              │
│   分布式系统不可能同时满足 C、A、P                          │
│   但可以满足 C+A (单点), C+P, A+P                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
