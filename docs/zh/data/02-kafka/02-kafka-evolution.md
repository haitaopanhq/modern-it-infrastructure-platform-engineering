# 06-02: 一图看懂 Kafka 演进史

## 核心观点
Kafka 从 LinkedIn 内部消息队列演变为分布式流平台标准。

## 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                    Kafka 架构                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   Producer ──► Topic ──► Partition ──► Consumer            │
│                     │         │            │                │
│                     │         │            │                │
│                ┌────┴────┐ ┌──┴──┐    ┌────┴────┐          │
│                │ Broker 1│ │  2  │    │ Broker N│          │
│                │ Leader  │ │Follower      │          │
│                └─────────┘ └──┬──┘    └─────────┘          │
│                               │                             │
│                      ┌────────┴────────┐                   │
│                      │   ZooKeeper    │                   │
│                      │ (KRaft in v3)  │                   │
│                      └─────────────────┘                   │
│                                                             │
│   核心概念:                                                 │
│   • Partition: 物理分组, 并行单位                           │
│   • Replica: 副本, 容灾                                     │
│   • Offset: 消费位置                                        │
│   • Consumer Group: 消费组, 负载均衡                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Kafka vs 传统 MQ

| 特性 | Kafka | RabbitMQ |
|------|-------|----------|
| 持久化 | 顺序写, 磁盘 | 内存/磁盘 |
| 顺序保证 | Partition 级别 | 队列级别 |
| 吞吐 | 百万/秒 | 万/秒 |
| 持久化 | 永久 | 可配置 |
