# 10-04: 一图看懂 AI Tool Protocol

## 核心观点
AI Tool Protocol 让 Agent 能够调用任意工具。

## 协议设计

```
┌─────────────────────────────────────────────────────────────┐
│                  AI Tool Protocol                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                   工具定义                           │  │
│   │   {                                                  │  │
│   │     "name": "kubectl_get_pods",                    │  │
│   │     "description": "获取集群 Pod 列表",            │  │
│   │     "parameters": {                                 │  │
│   │       "namespace": {"type": "string"},             │  │
│   │       "label": {"type": "string"}                  │  │
│   │     }                                               │  │
│   │   }                                                  │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                   调用流程                           │  │
│   │   1. LLM 理解用户请求                                │  │
│   │   2. 匹配可用工具                                    │  │
│   │   3. 提取参数                                        │  │
│   │   4. 执行工具                                        │  │
│   │   5. 返回结果                                        │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   安全考虑:                                                  │
│   • 工具白名单                                              │
│   • 参数验证                                                │
│   • 执行超时                                                │
│   • 审计日志                                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
