# IT 基础设施演进之路：从硬件、云原生到 AI Native

这不是一本按工具分类的教程。

如果把过去二十多年的基础设施演进真正串起来看，整个 IT 世界一直在围绕同一个问题反复变化：当业务越来越快、系统越来越大、团队越来越多、成本越来越高时，企业如何继续扩张，而不被自己的技术复杂度拖住。

很多技术名词后来被拆开讲：虚拟化、云计算、容器、Kubernetes、Service Mesh、GitOps、平台工程、可观测性、AI Infra、Agent Runtime。可它们并不是彼此割裂的技术清单，而是一条连续的技术史。每一代基础设施，表面上是在引入新工具，本质上是在重新分配抽象层、控制权、数据面和治理责任。

## 一条更长的演进路径

现代基础设施越来越像这样一条路径：

```text
Hardware
  -> Virtualization
  -> Cloud Control Plane
  -> Runtime
  -> Network
  -> Orchestration
  -> Platform Engineering
  -> AI Infra
  -> Agent Control Plane
```

最早的时代，基础设施的核心从来不是软件，而是硬件。CPU、内存、磁盘、网络带宽直接决定系统上限。互联网业务增长之后，企业第一次发现，限制业务扩张的不只是代码，而是资源无法被灵活利用。于是虚拟化出现了。它真正改变世界的地方，不只是多跑几台虚拟机，而是让计算资源脱离具体物理机器，变成可以分配、迁移、回收、调度的资源池。

后来这种抽象继续扩张。Ceph 把存储池化，Open vSwitch 把网络虚拟化，SR-IOV 与 DPDK 重新定义高性能网络路径，CSI、CNI、CRI 逐渐形成标准接口。硬件不再只属于某一台机器，而开始属于整个数据中心。

资源池化之后，一个更大的问题出现了：谁来管理这些资源。于是 Cloud Control Plane 到来。AWS、OpenStack、Harvester 这些平台真正重要的地方，不只是底层虚拟化，而是把基础设施变成统一控制平面。创建机器、分配网络、挂载存储、调度资源，从此进入 API 驱动时代。

Kubernetes 出现后，行业又进入一个新的中间层时代。它像一种分布式容器操作系统，统一了 Runtime 与 Orchestration 之间的关系。Mesos、Docker Swarm 逐渐退场，Kubernetes 成为事实标准。但任何成功到极致的中间层，最后都会暴露自己的边界。数据库跑 K8s，AI 跑 K8s，边缘跑 K8s，虚拟机跑 K8s，存储也跑 K8s，很多企业最后发现，自己管理的已经不是业务，而是 Kubernetes 本身。

AI Infra 正在重新打破这个结构。传统 Web Infra 关注 HTTP 请求、副本数、弹性扩缩容和服务治理；AI Infra 更关注 GPU 调度、KV Cache、显存拓扑、RDMA、Tensor Parallel、Pipeline Parallel、Token 延迟、多模型路由和推理吞吐。它们已经不是同一种问题。

所以 vLLM、Ray、TensorRT-LLM、SGLang、Triton 开始快速崛起。AI 时代并没有让硬件变得不重要，反而让 GPU、NVLink、HBM、RDMA、高速网络、Linux Kernel、eBPF、cgroups 重新成为系统上限。基础设施又回到那个最原始的问题：如何让硬件资源被更高效地利用。

## 本书的视角

这本书不把基础设施写成安装手册，而把它放回企业真实语境中理解。

| 视角 | 这本书关心的问题 |
|------|------------------|
| 技术 | 每一层抽象解决了什么系统边界问题 |
| 商业 | 业务增长、成本结构和交付速度如何推动架构变化 |
| 焦虑 | 团队为什么会害怕不可控、不可见、不可审计和不可回滚 |
| 复杂度 | 新工具为什么既降低复杂度，又制造新的复杂度中心 |
| 控制权 | 控制权如何从人转向脚本、控制面、Git、平台和 Agent |

已经发布的图文选题，正好构成了这本书的现实入口：`一图纵览IT基础设施演进之路` 讨论总体路径，`业务部门和基础 IT 部门，到底在关注什么？` 讨论组织语言差异，`一图读懂现代网络与协议` 讨论连接如何变成数据面，`一图读懂监控的前生今世` 讨论系统不可知带来的治理焦虑，`一图纵览账户与安全体系` 讨论安全边界如何从网络迁移到身份，`AI 会重新定义下一代基础设施吗？` 则把这条线推向 AI Native 与 Agent 控制面。

## 阅读方式

前七章是这本电子书的主干。

第 1 章先建立全局观：现代 IT 最大的问题已经不是功能，而是复杂度。第 2 章进入网络：连接不再只是把服务连起来，而是逐渐变成系统控制与数据面。第 3 章讨论数据库：业务增长、交易一致性、实时分析和 AI 检索如何不断重塑数据系统。第 4 章讨论存储：数据留存、访问模式、成本和 AI 训练让存储重新成为基础设施核心。第 5 章讨论可观测性：当系统变得不可直觉理解，监控就必须升级为治理能力。第 6 章讨论 DevOps 到平台工程：控制权如何从人工、脚本、流水线、GitOps 迁移到平台。第 7 章讨论平台工程核心能力：复杂度如何被产品化，最终变成组织可以复用的企业能力。

后续章节会继续把安全、运行时、故障治理、架构决策、实验环境、平台闭环和 Agent 工程接上这条主线。它们共同回答同一个问题：

> 基础设施演进不是工具越来越多，而是抽象、控制权、数据面和治理机制不断重组，最终把复杂系统转化为企业能力。
