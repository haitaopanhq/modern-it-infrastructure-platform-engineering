# 03-04: 一图看懂多云真正的问题

## 核心观点
多云不是技术问题，而是控制权和管理问题。

## 多云挑战

```
┌─────────────────────────────────────────────────────────────┐
│                    多云挑战                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   技术挑战:                管理挑战:                         │
│   ┌──────────────────┐   ┌──────────────────────────────┐  │
│   │  API 差异        │   │  统一认证和授权               │  │
│   │  网络打通        │   │  成本优化                     │  │
│   │  数据流动        │   │  运维统一                     │  │
│   │  性能一致        │   │  合规审计                     │  │
│   │  容灾切换        │   │  技能培养                     │  │
│   └──────────────────┘   └──────────────────────────────┘  │
│                                                             │
│   核心问题:                                                 │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  "你能带走什么？" - 云厂商锁定的实质                  │  │
│   │  • 数据格式                                         │  │
│   │  • 自动化脚本                                       │  │
│   │  • 培训知识                                         │  │
│   │  • 供应商关系                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │

```

## 解决方案

| 方案 | 工具 |
|------|------|
| 跨云编排 | Terraform, Pulumi |
| 统一监控 | Datadog, Grafana |
| 成本管理 | CloudHealth, Kubecost |
| 数据迁移 | Velero, Rclone |
