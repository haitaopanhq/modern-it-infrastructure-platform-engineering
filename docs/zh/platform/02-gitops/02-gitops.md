# 07-02: 一图看懂 GitOps

## 核心观点
GitOps 改变的不是发布方式，而是基础设施的控制权。

## GitOps 流程

```
┌─────────────────────────────────────────────────────────────┐
│                    GitOps 流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐            │
│   │  开发者   │───►│   Git    │───►│  ArgoCD  │            │
│   │  提交代码 │    │  (声明式)│    │  (同步)  │            │
│   └──────────┘    └──────────┘    └────┬─────┘            │
│                                        │                    │
│                                        ▼                    │
│                                 ┌──────────────┐            │
│                                 │   K8s 集群   │            │
│                                 │  声明式状态   │            │
│                                 └──────────────┘            │
│                                                             │
│   核心原则:                                                 │
│   1. 声明式配置 (Declarative)                               │
│   2. 版本控制 (Git as Single Source of Truth)              │
│   3. 自动同步 (Auto Sync)                                   │
│   4. 持续校验 (Continuous Verification)                     │
│                                                             │
│   工具: ArgoCD, Flux, Crossplane                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
