# 07-05: 一图看懂控制权迁移

## 核心观点
平台工程本质是控制权从运维向开发的迁移。

## 迁移图

```
┌─────────────────────────────────────────────────────────────┐
│                    控制权迁移                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   传统模式:                                                 │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐            │
│   │  开发    │───►│  提交代码 │───►│  运维    │            │
│   │          │    │          │    │  部署    │            │
│   └──────────┘    └──────────┘    └──────────┘            │
│        │                                    │               │
│        │         痛点:                      │               │
│        │         • 等待时间长               │               │
│        │         • 沟通成本高               │               │
│        │         • 责任不清                 │               │
│        │                                    │               │
│   平台工程模式:                                             │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐            │
│   │  开发    │───►│  平台    │───►│  基础设施│            │
│   │  自助    │    │  (自助)  │    │  (自动化)│            │
│   └──────────┘    └──────────┘    └──────────┘            │
│                                                             │
│   控制权变化:                                                │
│   运维控制 → 平台控制 → 开发者自助                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
