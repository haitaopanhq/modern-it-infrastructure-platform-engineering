# 07-03: 一图看懂 Platform Engineering

## 核心观点
平台工程不是工具团队，而是构建内部开发者体验。

## 平台架构

```
┌─────────────────────────────────────────────────────────────┐
│                  平台工程架构                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │              Internal Developer Portal              │  │
│   │  Backstage / Port / Select  │  服务目录 │ 文档     │  │
│   └─────────────────────────────┬───────────────────────┘  │
│                                 │                           │
│   ┌─────────────────────────────┼───────────────────────┐  │
│   │              Golden Path (标准化路径)               │  │
│   │   应用模板 │ 环境模板 │ 部署模板 │ CI/CD 模板       │  │
│   └─────────────────────────────┬───────────────────────┘  │
│                                 │                           │
│   ┌─────────────────────────────┼───────────────────────┐  │
│   │              自助服务能力                           │  │
│   │   环境创建 │ 资源申请 │ 权限审批 │ 监控配置         │  │
│   └─────────────────────────────┬───────────────────────┘  │
│                                 │                           │
│   ┌─────────────────────────────┴───────────────────────┐  │
│   │              基础设施抽象层                          │  │
│   │   Kubernetes │ Terraform │ Service Mesh │ CSI       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   平台团队职责:                                             │
│   • 构建 Golden Path (黄金路径)                            │
│   • 提供自助工具                                            │

│   • 沉淀最佳实践                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
