# 08-02: 一图看懂 OpenTelemetry

## 核心观点
OpenTelemetry 正在统一可观测性标准。

## OTel 架构

```
┌─────────────────────────────────────────────────────────────┐
│                 OpenTelemetry 架构                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                    SDKs                              │  │
│   │   Python │ Go │ Java │ Node.js │ 收集器             │  │
│   └───────────────────────────┬─────────────────────────┘  │
│                               │                             │
│                               ▼                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │              OTLP 协议 (OpenTelemetry Protocol)     │  │
│   │         统一数据传输格式 (Protobuf + JSON)          │  │
│   └───────────────────────────┬─────────────────────────┘  │
│                               │                             │
│           ┌───────────────────┼───────────────────┐         │
│           ▼                   ▼                   ▼         │
│   ┌─────────────┐     ┌─────────────┐     ┌─────────────┐   │
│   │ Prometheus  │     │    Jaeger   │     │    Loki     │   │
│   │  (指标)     │     │   (追踪)    │     │   (日志)    │   │
│   └─────────────┘     └─────────────┘     └─────────────┘   │
│                                                             │
│   核心概念:                                                  │
│   • Traces: 请求链路                                        │
│   • Metrics: 指标                                           │
│   • Logs: 日志                                              │
│   • Attributes: 标签                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
