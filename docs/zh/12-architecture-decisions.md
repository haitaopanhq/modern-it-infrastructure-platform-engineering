# 第 12 章：系统选型与架构决策

## 本章概述

本章介绍技术选型和架构决策的方法论，涵盖如何做技术选型、成本效益分析、决策树与 ADR、以及典型场景的选型建议。

## 12.1 如何做技术选型

### 选型流程

```
┌─────────────────────────────────────────────────────────┐
│                    技术选型流程                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   1. 需求分析                                           │
│      ├── 业务需求                                       │
│      ├── 非功能需求                                     │
│      └── 约束条件                                       │
│                                                         │
│   2. 市场调研                                           │
│      ├── 备选方案                                       │
│      ├── 社区活跃度                                     │
│      └── 生态成熟度                                     │
│                                                         │
│   3. 技术评估                                           │
│      ├── POC 验证                                       │
│      ├── 性能测试                                       │
│      └── 安全评估                                       │
│                                                         │
│   4. 决策                                               │
│      ├── 成本分析                                       │
│      ├── 风险评估                                       │
│      └── 管理层审批                                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 选型维度

| 维度 | 评估内容 | 权重 |
|------|----------|------|
| 功能匹配 | 能否满足业务需求 | 高 |
| 性能 | 吞吐量、延迟 | 高 |
| 成本 | 许可、云资源、人力 | 中 |
| 运维 | 复杂度、可靠性 | 中 |
| 团队 | 学习曲线、人员技能 | 低 |
| 未来 | 扩展性、社区发展 | 中 |

## 12.2 技术权衡分析

### 常见权衡

| 场景 | 权衡 | 建议 |
|------|------|------|
| 开源 vs 商业 | 成本 vs 支持 | 核心业务选商业 |
| 自建 vs 云服务 | 控制 vs 运维 | 差异化自建 |
| 成熟 vs 新技术 | 稳定 vs 创新 | 边缘验证 |
| 集中 vs 分布 | 简单 vs 扩展 | 渐进式演进 |

### 成本构成

```
总成本 (TCO) = 硬件成本 + 软件成本 + 人力成本 + 运维成本 + 培训成本 + 风险成本
```

| 成本类型 | 自建 | 云服务 |
|----------|------|--------|
| 初始成本 | 高 | 低 |
| 持续成本 | 波动 | 稳定 |
| 扩展成本 | 高 | 低 |
| 人员成本 | 高 | 低 |

## 12.3 决策树与 ADR

### 选型决策树示例：数据库

```
需要什么类型的数据库？
    │
    ├─ 需要事务？→ 否 → 向量数据库 / 搜索引擎
    │                    ├─ Milvus (AI 应用)
    │                    └─ Elasticsearch (搜索)
    │
    └─ 需要事务？→ 是 → 数据量 < 1TB？
                      │
                      ├─ 是 → MySQL / PostgreSQL
                      │          ├─ 简单 Web → MySQL
                      │          └─ 复杂查询 → PostgreSQL
                      │
                      └─ 否 → 分布式数据库
                                 ├─ 金融场景 → TiDB
                                 ├─ 全球分布 → CockroachDB
                                 └─ 分析为主 → ClickHouse
```

### ADR (Architecture Decision Record)

```markdown
# ADR-001: 使用 PostgreSQL 作为主数据库

## 状态
已通过

## 背景
需要为新微服务选择主数据库，需要支持：
- ACID 事务
- 复杂查询
- JSON 支持
- 高可用

## 决策
使用 PostgreSQL 15 作为主数据库

## 备选方案
- MySQL 8.0: 社区广泛，但复杂查询较弱
- MongoDB: JSON 友好，但事务支持有限
- TiDB: 分布式，但复杂度高

## 后果
### 正面
- 强大的 JSON 支持
- 丰富的索引类型
- 活跃社区

### 负面
- 需要 DBA 技能
- 复制配置复杂

## 评审日期
2026-05-11
```

## 12.4 典型场景选型

### 数据库选型

| 场景 | 推荐 | 备选 |
|------|------|------|
| Web 应用 | PostgreSQL | MySQL |
| 电商交易 | PostgreSQL | TiDB |
| 日志分析 | ClickHouse | Elasticsearch |
| 缓存 | Redis | Memcached |
| 消息队列 | Kafka | RabbitMQ |
| 向量检索 | Milvus | Pinecone |
| 图关系 | Neo4j | NebulaGraph |

### 存储选型

| 场景 | 推荐 | 说明 |
|------|------|------|
| 对象存储 | S3/MinIO | 海量非结构化 |
| 文件共享 | EFS/NFS | 多节点访问 |
| 块存储 | EBS/Longhorn | 数据库、高性能 |
| 混合存储 | JuiceFS | AI 训练 |

### 容器编排选型

| 规模 | 推荐 | 说明 |
|------|------|------|
| < 50 节点 | K3s | 轻量简单 |
| 50-500 节点 | K8s | 标准生产 |
| > 500 节点 | K8s + 多集群 | 大规模 |

### CI/CD 选型

| 场景 | 推荐 | 说明 |
|------|------|------|
| GitHub 用户 | GitHub Actions | 集成好 |
| GitLab 用户 | GitLab CI | 一体化 |
| 企业级 | Jenkins | 灵活强大 |
| K8s 部署 | ArgoCD | GitOps |

## 12.5 从局部最优到系统最优

### 避免局部最优

```
局部最优陷阱:
• 只看单点性能 → 整体性能差
• 选最便宜方案 → 运维成本高
• 追新技术的→ 技术风险大
• 满足当前需求 → 扩展性差
```

