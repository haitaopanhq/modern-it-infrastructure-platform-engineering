# 《图解 IT 技术演进史》系列规划

本目录用于沉淀公众号「行者深蓝」的一图类文章，并把它们改造成一本可以持续扩写的开源电子书。

这本书不按工具分类，也不写成教程。它按技术史的矛盾推进：单机时代的权限焦虑，网络时代的信任假设，云时代的资源抽象，AI 时代的算力重估，以及企业增长之后不可避免的稳定性、成本和控制权问题。

详细章节表见 [book-outline.md](./book-outline.md)。

## 全书主线

《图解 IT 技术演进史》的核心问题是：

> 为什么技术越先进，系统反而越复杂；为什么企业最后竞争的不是工具清单，而是把复杂系统长期稳定运行下去的能力。

全书围绕四条线索展开：

| 线索 | 关注的问题 |
|---|---|
| 技术演进 | 从单机、网络、虚拟化、云平台、平台工程到 AI 基础设施 |
| 商业压力 | 增长、成本、效率、稳定性、交付速度如何反向塑造技术架构 |
| 组织焦虑 | 权限、协作、故障、合规、跨部门边界如何变成基础设施问题 |
| 控制权迁移 | 控制权如何从人转向脚本、平台、控制面，最终走向 Agent |

## 写作规则

每篇文章必须回答一个现实问题，而不是解释一个工具名。

每章固定包含：

| 模块 | 要求 |
|---|---|
| 现实钩子 | 从业务增长、组织焦虑、成本压力或技术冲突开场 |
| 技术史脉络 | 说明这个问题为什么在当时出现 |
| 系统边界 | 解释技术解决了什么，又把什么复杂度转移给下一层 |
| 商业含义 | 回到成本、效率、稳定性、控制权或企业能力 |
| 一句话收束 | 回到“复杂系统如何变成企业能力”的主线 |
