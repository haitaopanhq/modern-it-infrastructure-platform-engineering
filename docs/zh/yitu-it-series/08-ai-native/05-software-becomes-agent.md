# 软件开始变成智能体

> AI Native 卷长文：从 LLM、AIGC、RAG 到 Agent Runtime，解释软件为什么开始从“功能集合”变成“长期运行的数字执行体”。

## 核心观点

如果把过去几年 AI 世界真正串起来看，一个非常明显的变化正在发生：

AI 正在从回答问题，逐渐进入执行任务。

最早的大模型时代，核心还是 LLM。大家讨论的是参数规模、上下文窗口、推理能力、训练数据、幻觉率。

后来进入 AIGC 时代，AI 开始生成文字、图片、代码、视频、音乐。但那个阶段，AI 本质上仍然只是被动生成器。

真正的分水岭，是 RAG、Memory、Tool Calling 与 AI Agent 出现之后。

因为 AI 第一次开始拥有长期记忆、工具调用、环境感知、状态管理、任务执行和跨系统协同能力。

于是 AI 开始从聊天框里的模型，逐渐变成长期运行的数字执行体。

## 从模型时代到 Agent Runtime 时代

这也是为什么最近整个行业开始出现一组新词：

- AI Agent
- MCP
- OpenClaw
- Hermes Agent
- Agent Runtime
- AI Gateway
- Autonomous Infra

这些词真正指向的，不只是“又一个 AI 应用”，而是一种新的运行形态。

行业真正竞争的，已经不再只是模型聪不聪明，而是：

> AI 能不能持续工作。

过去的软件世界，本质是人操作系统。人发起流程，人判断状态，人调用工具，人承担责任。系统负责执行，人负责决策。

Agent 时代开始第一次尝试让 AI 操作系统。

这意味着未来系统中长期运行的，不一定只是应用、容器、微服务，也可能是大量拥有记忆、能够调用工具、能够协同工作的 Agent。

## OpenClaw 与 Hermes Agent 的意义

OpenClaw、Hermes Agent 这类系统有意思的地方，不是它们又提供了一个聊天入口，而是它们开始尝试把 AI 从单次对话推进到持续运行的 Agent Runtime。

一个真正的 Agent Runtime 至少需要回答这些问题：

- 会话状态保存在哪里
- 长期记忆如何写入和召回
- 工具调用如何授权、审计和回滚
- 多轮任务如何恢复
- 多 Agent 如何分工与交接
- 成本、Token、失败和延迟如何被观测
- AI 代表谁行动，权限边界在哪里

从这个角度看，OpenClaw 的 Gateway、Agent Runtime、工具调用、记忆与执行链路，已经开始接近 AI 时代的控制平面雏形。

## MCP 让工具进入协议层

MCP 出现之后，事情进一步变化。

未来 AI 真正困难的问题，可能不是模型本身，而是上下文如何共享、工具如何标准化、Agent 如何协同、权限如何治理、状态如何同步。

如果每个工具都用自己的协议，每个 Agent 都维护自己的上下文，每套系统都重新定义权限和审计，那么 AI 生态很快会变成新的复杂度中心。

MCP / Tool Protocol 的意义，是把工具连接从应用内部实现，推向协议层。

这类似过去 API、RPC、Service Mesh 对软件系统的意义：它们不是消灭复杂度，而是让复杂度进入可治理的接口。

## AI 第一次进入基础设施层

更深层的变化是：AI 开始第一次真正进入基础设施层。

未来 GPU、存储、网络、日志、身份、安全、可观测性，都会围绕 AI 重构。

因为未来系统里真正持续运行的，可能不是人类写的一组固定服务，而是 AI 在运行 AI。

这也是为什么企业开始发现，未来真正重要的可能不是谁模型最大，而是谁拥有：

- 更强的 Agent Runtime
- 更低成本的推理基础设施
- 更长、更可控的记忆
- 更稳定的数据面
- 更安全的执行环境
- 更可治理的 AI Gateway
- 更清晰的 Agent 身份与权限模型

Cloud Native 没有结束。

它只是正在进入下一阶段：AI Native。

## 一图结构

| 阶段 | 软件形态 | 核心能力 |
|---|---|---|
| LLM | 回答问题 | 参数、上下文、推理能力 |
| AIGC | 生成内容 | 文本、图片、代码、视频 |
| RAG / Memory | 连接知识 | 检索、记忆、上下文管理 |
| Tool Calling | 调用系统 | API、工具、权限、审计 |
| AI Agent | 执行任务 | 状态、计划、任务恢复 |
| Agent Runtime | 长期运行 | 会话、记忆、工具、成本、观测 |
| AI Native Platform | 智能体基础设施 | AI Gateway、MCP、多 Agent 调度 |

## 一句话总结

过去二十年的基础设施演进，本质上一直在解决如何让系统越来越自动化。

AI Agent 可能会把这件事推进到下一阶段：

> 系统越来越复杂，于是人类把控制权交给平台；平台越来越复杂，于是人类开始把控制权交给 AI。

最后一句，也送给整个系列：

> 这世界唯一不变的，就是一切都在变化。
