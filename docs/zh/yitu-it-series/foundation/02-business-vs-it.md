# 业务部门和基础 IT 部门，到底在关注什么？

> **一个想快，一个怕死**

## 核心冲突

很多企业内部最大的冲突，不是"技术不行"，也不是"业务不懂技术"——而是大家盯着的根本不是同一个世界。

| 维度 | 业务部门 | 基础 IT 部门 |
|------|---------|-------------|
| 核心指标 | PV、UV、DAU、转化率、GMV | SLA、SLO、Error Budget、成本 |
| 关注点 | 增长、收入、用户体验 | 稳定性、资源池、风险边界 |
| 系统视角 | 结果 | 链路 |
| 常用语言 | "系统卡了"、"用户流失" | "P99 延迟"、"连接池耗尽"、"GPU 等 IO" |
| 深层焦虑 | 市场窗口关闭 | 系统失控 |

## 金句

> **业务看到的是结果，IT 看到的是链路；成熟组织要做的，就是把结果和链路重新接起来。**

## 为什么冲突越来越严重

过去单体时代，一个网站卡了，可能只是单机 CPU 满了。今天一个用户点击失败，背后跨越：CDN → WAF → API Gateway → K8s → Service Mesh → Redis → MQ → 数据库 → 对象存储 → GPU 推理 → 第三方 API → 支付网关 → 日志系统 → 监控系统。

业务看到的是"用户流失"，SRE 看到的是 Error Budget 消耗，Infra 看到的是资源池容量，管理层看到的是预算。**每个人都只看到一部分，却都以为自己看到了全局。**
