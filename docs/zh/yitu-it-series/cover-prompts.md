# 《现代 IT 基础设施演进史》封面图提示词

## 总封面

低信息密度封面图，主题为“现代 IT 基础设施演进史”。画面使用一条从左到右的基础设施演进路径，不堆工具清单，只保留 7 个大阶段：

```text
Hardware -> Virtualization -> Cloud Control Plane -> Kubernetes -> Platform Engineering -> AI Infra -> AI Native
```

视觉上每个阶段用一个清晰图标表达：服务器、资源池、云控制面、编排网格、平台控制台、GPU 集群、AI Agent。背景是简洁的数据中心与云平台轮廓，右侧逐渐过渡到 AI Agent 控制面。整体像技术史书籍封面，不要密集小字，不要复杂架构图，不要营销海报感。

风格：深蓝、白色、少量青色高光，克制、专业、图示为主。

## 章节封面原则

| 原则 | 要求 |
|---|---|
| 信息密度 | 每张封面最多 3 到 5 个大节点 |
| 文本 | 只保留章节标题、1 条演进路径、少量关键词 |
| 图形 | 用路径、层级、控制面、数据流表达，不做复杂系统架构图 |
| 色彩 | 保持深蓝、白、青色高光，避免花哨渐变 |
| 主题 | 必须能一眼看出“基础设施演进”，而不是某个工具教程 |

## AI Infra 封面

低信息密度封面图，主题为“Kubernetes 下沉，AI Infra 上升”。画面中 Kubernetes 作为中间层底座逐渐下沉，右侧上方出现 AI Gateway、GPU Scheduler、Inference Fabric、Agent Runtime 四个大模块。底部是 GPU / RDMA / NVMe / Linux Kernel 的简化硬件底座。不要展示密集工具名，不要把 vLLM、Ray、Triton 都堆进去，只表现“Cloud Native -> AI Native”的结构变化。
