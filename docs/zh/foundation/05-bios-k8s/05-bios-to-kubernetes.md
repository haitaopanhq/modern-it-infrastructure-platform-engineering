# 01-05: 一图看懂从 BIOS 到 Kubernetes

## 核心观点
从 BIOS 到 K8s 是基础设施控制权的转移史。

## 完整演进路径

```
┌─────────────────────────────────────────────────────────────┐
│              基础设施控制权迁移                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  BIOS/UEFI → Hypervisor → Cloud → Container → Platform     │
│     │           │           │         │          │          │
│     ▼           ▼           ▼         ▼          ▼          │
│  硬件初始化  虚拟机管理   云控制   容器编排   平台工程     │
│                                                             │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ 控制权迁移:                                            ││
│  │                                                         ││
│  │ 1. BIOS: 硬件厂商控制                                  ││
│  │ 2. Hypervisor: 系统管理员控制                         ││
│  │ 3. Cloud: 云厂商控制                                  ││
│  │ 4. Container: 开发者获得部分控制                      ││
│  │ 5. Platform: 平台团队统一管理                         ││
│  │                                                         ││
│  │ AI 时代: Agent 控制面出现                              ││
│  └─────────────────────────────────────────────────────────┘│
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 各层职责

| 层级 | 控制内容 | 角色 |
|------|---------|------|
| BIOS | 硬件初始化、启动顺序 | 设备厂商 |
| Hypervisor | 虚拟机生命周期 | 虚拟化管理员 |
| Cloud | 资源分配、计费 | 云厂商 |
| Kubernetes | 容器编排、服务治理 | 平台团队 |
| Platform | 应用交付、自助服务 | 开发者 |
