# 01-01: 一图看懂 Linux 为什么从未离开基础设施核心

## 核心观点
Linux 从未离开基础设施核心，它只是从"可见"变成"隐形"。

## 演进图

```

│                           │                │                │
```

## 关键知识点

| 组件 | 作用 | 平台工程关联 |
|------|------|-------------|
| cgroups | 资源限制 | K8s 资源配额基础 |
| namespace | 隔离 | 容器隔离基础 |
| System Call | 系统接口 | 安全审计点 |
| eBPF | 内核可编程 | 可观测性/网络 |

## AI 时代的 Linux

- **GPU 驱动**：CUDA/NVIDIA 驱动运行在 Linux
- **DPU**：数据处理器需要 Linux 作为控制平面
- **容器安全**：内核漏洞直接影响容器安全
- **边缘计算**：Linux 是边缘设备主流 OS
