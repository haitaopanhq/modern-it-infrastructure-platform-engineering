# 05-03: 一图看懂 API Gateway 演进

## 核心观点
API Gateway 从反向代理演进为流量网关和 API 管理平台。

## 演进图

```
┌─────────────────────────────────────────────────────────────┐
│                  API Gateway 演进                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   第一代: 反向代理                                           │
│   Nginx, Apache → 负载均衡, 静态资源                        │
│                                                             │
│   第二代: API 网关                                          │
│   Kong, Traefik → 路由, 限流, 认证                         │
│                                                             │
│   第三代: 流量网关                                          │
│   Envoy, Istio → 流量管理, 熔断, 灰度                      │
│                                                             │
│   第四代: API 管理                                          │
│   Apigee, AWS API GW → 全生命周期管理                       │
│                                                             │
│   AI 时代: AI Gateway                                       │
│   ────────────                                              │
│   模型路由, Token 控制, 多模型切换                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
