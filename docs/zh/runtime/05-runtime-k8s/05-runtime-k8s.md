# 04-05: 一图看懂 Runtime 与 Kubernetes 的关系

## 核心观点
Kubernetes 通过 CRI 抽象 Runtime，实现多 Runtime 支持。

## CRI 架构

```
┌─────────────────────────────────────────────────────────────┐
│                    Kubernetes CRI                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │              kubelet                                 │  │
│   │   ┌──────────────────────────────────────────────┐  │  │
│   │   │              CRI gRPC                         │  │  │
│   │   │   • RunPodSandbox                            │  │  │
│   │   │   • CreateContainer                          │  │  │
│   │   │   • StartContainer                           │  │  │
│   │   │   • PullImage                                │  │  │
│   │   └──────────────────────────────────────────────┘  │  │
│   └───────────────────────────┬──────────────────────────┘  │
│                               │                              │
│           ┌───────────────────┼───────────────────┐         │
│           ▼                   ▼                   ▼         │
│   ┌─────────────┐     ┌─────────────┐     ┌─────────────┐   │
│   │  containerd │     │    CRI-O    │     │  Docker     │   │
│   │  (默认)     │     │   (Podman)  │     │  (遗留)     │   │
│   └─────────────┘     └─────────────┘     └─────────────┘   │
│           │                   │                   │         │

│                    ┌─────────────────────┐                  │

│                    │   (容器运行)        │                  │
│                    └─────────────────────┘                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
