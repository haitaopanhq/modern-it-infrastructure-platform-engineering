# 02-05: 一图看懂 Hypervisor 演进

## 核心观点
Hypervisor 从 Type 1/2 到云原生，形态不断变化。

## 类型对比

```
┌─────────────────────────────────────────────────────────────┐
│                 Hypervisor 类型                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   Type 1 ( Bare Metal )         Type 2 ( Hosted )          │
│   ┌─────────────────┐           ┌─────────────────┐         │
│   │   ESXi/KVM     │           │   VMware WS    │         │
│   │   直接运行在   │           │   运行在 OS    │         │
│   │   硬件上       │           │   上           │         │
│   │   性能高       │           │   性能低       │         │
│   │   生产环境     │           │   开发测试     │         │
│   └─────────────────┘           └─────────────────┘         │
│                                                             │
│   云原生虚拟化:                                              │
│   ┌─────────────────────────────────────────────────────┐  │
│   │   Firecracker (AWS)  │  gVisor (Google) │ Kata     │  │
│   │   轻量虚拟机         │  用户空间内核     │  安全容器│  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 云原生虚拟化

| 项目 | 类型 | 特点 |
|------|------|------|
| Firecracker | MicroVM | 轻量、快速启动 |
| gVisor | Sandboxed | 隔离用户空间内核 |
| Kata Containers | Secure Container | 硬件虚拟化隔离 |
| Cloud-Hypervisor | Rust VMM | 现代实现 |
